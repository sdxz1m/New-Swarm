# 当前术语来源审计

来源文件：

- `docs/localization/g1tyx-chs-extracted.md`
- `docs/localization/g1tyx-chs-extracted.json`

说明：

- `精确`：汉化表中存在可直接对应的独立词条或明确 UI 词条。
- `短语推断`：汉化表中没有独立单位名词条，只在描述、成就或正则句子中出现，需要人工确认后再作为单位名使用。
- `待确认`：当前显示词不应继续无标注使用。

## 当前重点问题

| Old 名称 | 当前显示 | 来源状态 | 汉化表依据 |
| --- | --- | --- | --- |
| `empress` / `hive empress` | 主巢女皇 | 短语推断 | `Hive empresses produce more hive queens.` -> `主巢女皇可以产生更多主巢虫后。`；`Grow your first hive empress` -> `首次产生主巢女皇`；`/^Grow (.+) hive empresses$/` -> `产生$1单位 主巢女皇` |

结论：`主巢女皇` 不是汉化表里的独立单位名词条，不能再称作“直接来自单位名汉化”。它只是从相关短语中推断出的显示名。后续应优先使用精确词条；没有精确词条时，必须在迁移记录中标注为短语推断或待确认。

## 后续规则

1. 单位名优先找独立词条，例如 `queen`、`territory`、`hive mind` 这类。
2. 没有独立词条时，可以参考成就、正则和描述里的固定译法，但必须标注为“短语推断”。
3. 如果同一英文在不同短语里有不同译法，先不写入 `src/data`，列为待确认。
4. 后续新增单位时同步更新本文件，避免把推断翻译误说成精确翻译。
